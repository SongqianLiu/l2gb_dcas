L2GB ADRS - 避免驾驶员误用​
This article discusses the requirements of the L2GB ADRS system to avoid driver misuse, including system state definitions, design requirements, trigger conditions, and test suggestions. Key points include:​
System state definitions : The system has four states: active state (the system or function assists the driver in controlling the vehicle's lateral and longitudinal movements), standby state (the system or function does not perform any lateral or longitudinal motion control), on state (the system or function can be used to assist the driver, including standby or active states), and off state (the system is prevented from assisting the driver).​
Design to prevent misuse : The system's design should effectively handle foreseeable driver misuse, and the design of activation and exit operations should prevent such misuse.​
Preventing false activation : When detecting beyond the system's ODD boundary or when the driver accidentally presses the Hero button, the system should not be activated and should issue an obvious driver reminder.​
Test conditions for false intervention : In daytime, on a long - straight road with at least one - way two - lane and a dotted middle lane line, the test vehicle activates the system while cruising at Vsmaxset, then accelerates to (upper limit of design operating speed range + 5) km/h.​
Test end conditions : The test ends when either the system of the test vehicle exits or the vehicle maintains a speed of (upper limit of design operating speed range + 5) km/h for 5s.​
Passing conditions : The system should exit the active state before the end of the test, and the HMI prompt should match the "User Manual" and include a continuous optical prompt signal.​
Trigger conditions list : There are trigger conditions such as false intervention, misuse after activation, and false activation. ​
