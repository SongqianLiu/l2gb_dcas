L2GB ADRS - HMI（General）​
This article discusses the overall HMI requirements for the L2GB system, including system status prompts, driver identity confirmation, 30-day user instruction confirmation, and time gap prompts, along with corresponding test suggestions. Key points include:​
System status prompts : The system should provide clear and sufficient system status and motion control information to the driver, with optical signals being clearly visible even during the day.​
Driver identity confirmation : After each vehicle power-on/ignition (excluding engine auto - start/stop), at least one of the following methods should be used to confirm if the driver has completed system training: system training confirmation, biometric recognition, or account login when the vehicle is stationary.​
30 - day user instruction confirmation : The vehicle should provide a way for the driver to confirm reading and understanding the system instructions. Confirmation is only allowed when the vehicle is stationary, using methods like long - press or at least two purposeful operations. If the instructions change due to software upgrades or the driver hasn't confirmed in 30 days, confirmation is required before the next system activation.​
Time gap prompts : The system should continuously prompt the current following time gap to the driver through optical signals or prompt the time gap setting when the system is activated or the setting changes.​
Time gap prompt test : During road tests, confirm the time gap prompt method, which should be consistent with the "user instructions" and follow one of the methods in [4.6.1.11](4.6.1.11).​
Driver identity confirmation test : When an untrained driver tries to activate the system, the function activation should be suppressed.​
30 - day user instruction confirmation test : When the system instructions change or the driver hasn't confirmed in 30 days, a confirmation prompt should pop up when the vehicle is stationary. If the user doesn't confirm, the system activation is suppressed. ​
