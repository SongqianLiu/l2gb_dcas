L2GB ADRS - EOR​
This article discusses the requirements, functions, timing, HMI requirements, recovery conditions, and test suggestions for the L2GB ADRS - EOR system. Key points include:​
System Design Requirement : The system should ensure the driver continuously performs dynamic driving tasks, and detect the driver's eye - gaze deviation from the driving - related area.​
Detection Method : If direct eye - gaze detection is unavailable, the system can indirectly detect the driver's head posture.​
Speed Condition : When the system is activated and the speed is over 10 km/h, it should use hand - off and eye - gaze - off detection to monitor the driver.​
Function Linkage : The system should actively start eye - gaze - off detection after activation. If it's unavailable, the function should not be activated.​
Timing Requirement : When the speed is over 10 km/h, the system should issue an EOR within 5 s after the driver's eye - gaze deviates, and upgrade it within 3 s if the deviation continues.​
HMI Requirement : EOR should include optical, acoustic or tactile signals; DCA should clearly prompt the driver to regain lateral control.​
Test Suggestion : Tests include daytime and nighttime eye - gaze - off tests and DMS occlusion tests with specific test conditions and passing criteria. ​
